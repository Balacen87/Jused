## Игра: **«Судебный процесс»**

**Жанр:** логическая / детективная / текстовый симулятор
**Платформа:** браузер (HTML + JS)
**Графика:** минимальная — карточки, текст, иконки
**Цель:** игрок выступает судьёй и выносит решение на основе доказательств.

---

# 1. Основная концепция

Игрок получает **уголовное или гражданское дело**.

Он должен:

1. изучить доказательства
2. выслушать показания свидетелей
3. обнаружить противоречия
4. оценить достоверность информации
5. вынести вердикт

Каждое дело имеет **одну или несколько возможных истин**.

---

# 2. Игровой цикл

### Шаг 1 — получение дела

Игрок получает карточку:

```
Дело №1284

Подсудимый: Иван Петров
Обвинение: кража телефона
Место: кафе
Дата: 12 мая
```

---

### Шаг 2 — изучение материалов

Игрок открывает вкладки:

* обвинение
* защита
* улики
* свидетели
* экспертизы

Пример интерфейса:

```
[Обвинение]
[Защита]
[Свидетели]
[Улики]
[Экспертиза]
[Вердикт]
```

---

# 3. Типы доказательств

### 1. Свидетельские показания

```
Свидетель: Анна
"Я видела, как Петров взял телефон со стола."
```

Но другой свидетель говорит:

```
"Петров сидел рядом со мной и никуда не вставал."
```

Игрок должен **выявить конфликт показаний**.

---

### 2. Видео или фото

Минимальная графика — просто описание.

```
Камера наблюдения:

В 14:32 человек в красной куртке
подходит к столу.
```

Но:

```
Подсудимый был в синей куртке.
```

---

### 3. Экспертиза

```
Отпечатки пальцев:
найдены на телефоне.
```

Но:

```
Подсудимый утверждает,
что раньше держал телефон владельца.
```

---

### 4. Вещественные доказательства

```
Телефон найден у обвиняемого.
```

Но:

```
Он говорит, что нашел его на улице.
```

---

# 4. Механика анализа

Игрок может **отмечать факты**.

Например:

```
[ ] противоречие
[ ] подозрительно
[ ] подтверждено
```

Игра может показывать **схему связей**.

```
Свидетель А
      |
      | противоречит
      |
Свидетель Б
```

---

# 5. Система доверия

Каждый свидетель имеет параметр:

```
достоверность: 0–100
```

На неё влияют:

* судимость
* родственные связи
* противоречия
* алкоголь
* стресс

---

# 6. Возможные действия игрока

Игрок может:

* задать вопрос свидетелю
* запросить экспертизу
* вызвать нового свидетеля
* проверить алиби
* назначить очную ставку

Пример:

```
Вы хотите задать вопрос?

1 — Где вы стояли?
2 — Какого цвета была куртка?
3 — Вы уверены?
```

---

# 7. Очная ставка (очень интересная механика)

```
Свидетель Анна
Свидетель Сергей
```

Игрок задаёт вопрос:

```
Кто находился возле стола?
```

Ответы:

```
Анна: Петров
Сергей: неизвестный мужчина
```

Игра фиксирует **противоречие**.

---

# 8. Финал дела

Игрок должен вынести решение.

```
ВЕРДИКТ

[ ] виновен
[ ] невиновен
[ ] недостаточно доказательств
```

---

# 9. Система очков

Игрок получает:

### За правильное решение

+100

### За обнаруженные противоречия

+10

### За ложное обвинение

−50

---

# 10. Карьера судьи

Игрок проходит путь:

```
мировой судья
↓
районный суд
↓
областной суд
↓
верховный суд
```

Дела становятся сложнее.

---

# 11. Генерация дел (очень важно)

Дела могут генерироваться процедурно.

Параметры:

```
тип преступления
свидетели
улики
ложные показания
алиби
```

Например:

```
убийство
кража
мошенничество
драка
ДТП
```

---

# 12. Пример одного дела

### Ситуация

```
Кража кошелька в автобусе
```

### Улики

```
кошелек найден у обвиняемого
```

### Свидетель

```
"Он украл кошелек."
```

### Но

```
на кошельке нет его отпечатков
```

### Поворот

```
свидетель — родственник потерпевшего
```

---

# 13. Особые события

Иногда происходят:

* лжесвидетельство
* поддельная экспертиза
* давление СМИ
* коррупция
* признание вины

Игрок решает:

```
принять признание
или проверить доказательства
```

# 15. Почему игра может стать популярной

2. простая механика
3. короткие партии
4. моральный выбор
5. процедурная генерация дел
