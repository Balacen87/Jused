## Система лжи свидетелей для игры **«Судебный процесс»**

Цель системы — чтобы показания свидетелей были **реалистичными**, а не просто случайной ложью. В реальных судах ложные показания возникают по нескольким причинам:

1. **намеренная ложь**
2. **ошибка памяти**
3. **предвзятость**
4. **ограниченное наблюдение**
5. **стресс**
6. **социальное давление**

Поэтому система должна моделировать **не только ложь**, но и **искажение информации**.

---

# 1. Общая архитектура системы лжи

```text
WitnessTruthEngine
│
├ Witness Profile
├ Observation Model
├ Memory Distortion
├ Bias System
├ Intentional Lie System
└ Statement Generator
```

---

# 2. Профиль свидетеля

Каждый свидетель имеет параметры.

```typescript
type Witness = {
  id: string
  credibility: number
  memoryAccuracy: number
  stressLevel: number
  bias: number
  relationToDefendant: string
  relationToVictim: string
  lyingProbability: number
}
```

Пример:

```json
{
  "credibility": 0.75,
  "memoryAccuracy": 0.6,
  "stressLevel": 0.4,
  "bias": 0.3,
  "lyingProbability": 0.1
}
```

---

# 3. Какие факты может видеть свидетель

Свидетель не должен знать всё.

Каждый получает **частичное наблюдение**.

```text
Observation
│
├ location
├ action
├ object
├ appearance
└ time
```

Например:

```text
Witness A
видел человека у стола
но не видел лицо
```

---

# 4. Основные типы искажений

Система должна генерировать **5 типов показаний**.

| тип        | описание                    |
| ---------- | --------------------------- |
| true       | полностью правдивое         |
| incomplete | неполное                    |
| mistaken   | ошибка памяти               |
| biased     | искажено из-за предвзятости |
| lie        | сознательная ложь           |

---

# 5. Алгоритм выбора типа показаний

Псевдокод:

```javascript
function determineStatementType(witness){

    r = random()

    if(r < witness.lyingProbability)
        return "lie"

    if(r < witness.memoryAccuracy)
        return "true"

    if(r < witness.memoryAccuracy + 0.2)
        return "mistaken"

    return "incomplete"
}
```

---

# 6. Генерация правдивых показаний

Если свидетель говорит правду:

```text
TrueStatement = observation факт
```

Пример:

```text
Я видел, как человек взял телефон со стола.
```

---

# 7. Ошибка памяти

Люди часто путают:

* цвет одежды
* точное время
* лицо
* расстояние
* последовательность событий

### Алгоритм

```javascript
function distortFact(fact){

    switch(fact.type){

        case "color":
            return randomSimilarColor()

        case "time":
            return fact.time + random(-3,+3) minutes

        case "appearance":
            return "someone similar"

        case "action":
            return "picked something"
    }

}
```

Пример:

Истина:

```text
человек в синей куртке
```

Показание:

```text
человек в темной куртке
```

---

# 8. Неполное показание

Свидетель говорит только часть.

Истина:

```text
человек взял телефон и положил в карман
```

Показание:

```text
человек что-то взял со стола
```

---

# 9. Предвзятость

Если свидетель:

* знаком с потерпевшим
* конфликтует с подсудимым

его показания могут смещаться.

Пример:

```text
relationToVictim = friend
bias = 0.5
```

Алгоритм:

```javascript
if(witness.bias > 0.4)
    statement.confidence += exaggeration
```

Пример:

Истина:

```text
Я видел, как он подошёл к столу
```

Предвзятое показание:

```text
Я уверен, что он взял телефон
```

---

# 10. Намеренная ложь

Ложь должна иметь **мотив**.

Типы мотивов:

```text
protect friend
hate defendant
fear
money
self-protection
```

Пример:

```text
Свидетель — друг подсудимого.
```

Он говорит:

```text
Я не видел его там.
```

Хотя видел.

---

# 11. Алгоритм генерации ложного показания

```javascript
function generateLie(truthFact){

    type = random([
        "deny",
        "invert",
        "replace_person",
        "replace_time"
    ])

    switch(type){

        case "deny":
            return "I did not see anything"

        case "replace_person":
            return "another man took the phone"

        case "replace_time":
            return "it happened later"

    }

}
```

# 12. Поведенческие признаки лжи

Чтобы игрок мог анализировать свидетеля, можно добавить:

```text
нервозность
уклончивость
изменение формулировок
несоответствие деталей
```

Пример:

```text
"Ну… кажется… возможно это был он…"
```

---

# 13. Очная ставка

Когда два свидетеля дают разные показания.

Алгоритм:

```javascript
function crossExamination(witnessA, witnessB){

    if(conflict(witnessA.statement, witnessB.statement))
        return "contradiction detected"

}
```

---

# 14. Эволюция показаний

При повторном допросе свидетель может **изменить детали**.

Пример:

Первое показание:

```text
Он был в красной куртке
```

Второе:

```text
Может быть в темной
```

Это создаёт реализм.

Каждое показание имеет параметр:

```typescript
statementConfidence
```

Пример:

```text
0.9 — уверен

0.3 — не уверен
```

# 16. Как игрок выявляет ложь

Игрок анализирует:

* противоречия
* уверенность
* мотив
* совпадение с уликами

# 17. Система вероятности лжи

Формула:

```text
lieChance =
baseLieProbability
+ bias
+ relationFactor
+ fear
```

Пример:

```text
0.1 + 0.3 + 0.2 = 0.6
```

---

# 18. Детектор противоречий

Алгоритм:

```javascript
function checkContradiction(statementA, statementB){

    if(statementA.time != statementB.time)
        return true

    if(statementA.color != statementB.color)
        return true

    if(statementA.person != statementB.person)
        return true
}
```

---

# 19. Баланс системы

Чтобы игра была честной:

* не более **30% лживых свидетелей**
* большинство — **ошибки памяти**

Реалистичная модель:

```text
truth 50%
mistake 30%
incomplete 10%
lie 10%
```

# 20. Пример полного сценария

Истина:

```text
Иван взял телефон в 14:32
в синей куртке
```

Свидетель А:

```text
Я видел, как он взял телефон
```

Свидетель B:

```text
Мне показалось, что это был другой мужчина
```

Свидетель C:

```text
Он был в красной куртке
```

Игрок должен понять:

* свидетель C ошибся
* свидетель B видел плохо
* свидетель A прав

# 21. Почему такая система работает

Она создаёт:

* реалистичные показания
* логические противоречия
* возможность анализа
* ощущение настоящего суда

---

# 22. Самое важное правило

**Ложь должна быть объяснима.**

Если свидетель врёт, игрок должен иметь шанс понять:

* **почему**
* **как**
* **в чём именно**
